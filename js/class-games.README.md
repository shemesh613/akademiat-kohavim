# class-games.js — פאץ' אינטגרציה ל-index.html

קובץ זה (`js/class-games.js`) לא נוגע ב-`index.html`. הפעולות הבאות הן על המתאם.

## 1. טעינת הסקריפט

להוסיף תג script ליד שאר קבצי הסוכנים (למשל ליד `island-engine.js`), בכל מקום ב-`<head>`/סוף
`<body>` — סדר הטעינה ביחס ל-`window.AK` **לא קריטי**: כל הפונקציות הציבוריות של `ClassGames`
בודקות את `window.AK` runtime (בזמן קריאה בפועל), לא בזמן טעינת הקובץ.

```html
<script src="js/class-games.js"></script>
```

## 2. חיבור ה-hook על `addPoints` (השורה הקריטית היחידה)

ב-`index.html`, בתוך `function addPoints(studentId, amount, fromX, fromY) { ... }` (מתחיל בשורה
~2507), יש להוסיף שורה אחת **מיד אחרי** עדכון הנקודות בפועל — כלומר אחרי:

```js
s.points = Math.max(0, s.points + amount + comboBonus);
s.weeklyPoints = (s.weeklyPoints || 0) + amount + comboBonus;
```

להוסיף שם (לפני `awardStickersForPoints` או אחריו, לא משנה — רק אחרי ששני השדות למעלה כבר עודכנו):

```js
if (window.ClassGames && typeof window.ClassGames.onPointsAdded === 'function') {
  try { window.ClassGames.onPointsAdded(s, amount + comboBonus); } catch (e) {}
}
```

**חשוב:** להעביר `amount + comboBonus` (הסכום בפועל שנוסף, כולל בונוס קומבו אם היה), לא את
`amount` הגולמי בלבד — כך שקופת הזמן היומית וההגרלה המשוקללת של "מי משחק/ת היום" משקפות את מה
שבאמת נצבר.

`onPointsAdded` **לא** נוגע ב-`student.points` ו**לא** ממיר בעצמו נקודות ל-`island.coins` — התפקיד
היחיד שלו הוא לעדכן מונים פנימיים של `class-games.js` (`klass.games.dailyPoints` /
`dailyPointsByStudent`) לצורך נוסחת "שניות המשחק" ובחירת השחקן/ית היום. **המרת 10 נקודות תלמיד = 1
אבן בניין (SPEC §3) היא hook נפרד ואחר** שכבר אמור להיות קיים/מתוכנן בנפרד (ראו SPEC "כבר מוגדר
ב-SPEC כ-hook על addPoints — לא לגעת") — לא לבלבל בין השניים. בונוס הקוינים שהמשחקים עצמם מזרימים
ל-`island.coins` (x1/x1.5/x2 בסיום מיני-משחק) הוא ערוץ **נוסף ונפרד** לגמרי, מנוהל כולו בתוך
`class-games.js` ב-`renderCelebrate`.

## 3. איך מפעילים את המסך

אין בקובץ שלי שום כפתור UI (לא נגעתי ב-`index.html`). המתאם צריך להוסיף איפשהו (מסך מורה / מסך
פרויקטור) כפתור/טריגר שקורא:

```js
window.ClassGames.open();
```

זה יבחר לבד את משחק היום (רוטציה לפי יום בשבוע, שישי-שבת = מסיבת קונפטי), יבחר לבד את השחקן/ית
(אלגוריתם 50/50), יחשב לבד כמה שניות הכיתה הרוויחה, ויריץ את כל הרצף (הכרזה → משחק → חגיגה) עד
`window.ClassGames.close()` אוטומטי בסוף. אין צורך בשום קלט תוכן מהמורה.

לבדיקות/QA אפשר לכפות משחק ספציפי בלי לחכות ליום הנכון בשבוע:

```js
window.ClassGames.open({ game: 'meteor' });     // 'meteor' | 'treasure' | 'colorrush' | 'confetti'
```

## 4. מצב "הפעלה עצמית" (self-driving, אופציונלי)

הקובץ כבר כולל `setInterval` פנימי שבודק כל 20 שניות אם **גם** מתקיים `?projector=1` ב-URL **וגם**
לא שוחק עדיין משחק היום (`klass.games.dailyDone===false`) **וגם** השעון הגיע לשעה המוגדרת
(ברירת מחדל `12:15`) — ואז קורא ל-`open()` לבד, בלי שאף אחד לוחץ כלום. אפשר לשנות את השעה:

```js
window.ClassGames.config.autoTime = '12:30';
```

אם לא רוצים הפעלה עצמית בכלל — פשוט לא להוסיף `?projector=1` לכתובת של מסך הפרויקטור, וזה לעולם
לא יופעל לבד (רק ידנית דרך `open()`).

## 5. שדות ה-state שנוספו ל-`klass.games` (מעבר לסכימה הבסיסית ב-SPEC)

כל השדות הבאים הם בבעלות `class-games.js` בלבד, מאותחלים אוטומטית בהגנה מלאה
(`klass.games = klass.games || {}`), בטוחים להתעלמות מכל קוד אחר:

| שדה | תפקיד |
|---|---|
| `dailyPoints` / `dailyPointsByStudent` | נקודות שנצברו היום (סה"כ / לפי תלמיד) — מתאפס כל חצות |
| `dailyDate` | תאריך היום הנוכחי (לזיהוי חצות/rollover) |
| `lastPlayedByStudent` | timestamp אחרון שכל תלמיד/ה שיחק/ה — להגרלת ההוגנות |
| `playedSet` | מי כבר שיחק/ה במחזור הנוכחי — מתאפס כשכולם שיחקו |
| `weekBonusDate` / `weekStartCoinsSnapshot` | לסיכום השבועי במסיבת הקונפטי |

השדות מה-SPEC (`lastPlayed`, `totalScore`, `plays`, `dailyDone`) נשמרים כפי שהוגדרו.

## 6. מה מומש בפועל (4 משחקים + תשתית, מתוך 10 הרעיונות ב-RESEARCH)

- **☄️ מרוץ המטאור** — ריצה אינסופית, קפיצה בקליק/רווח/מגע, מכשולים גדלים במספר ובקצב לפי
  הזמן שהוקצה (3 ב-20 שנ' → 12 ב-60 שנ'), שיא מובנה קרוב לסוף.
- **💰 ניחוש הקופה** — הכיתה צועקת מספר בקול, התורן מקליד ניחוש במקלדת-מסך/מקלדת, רמזים
  מצטמצמים מוצגים רק במשחקים ארוכים, ניקוד לפי קרבה למספר האמיתי.
- **🌈 מירוץ הצבעים** — 4 ריבועי צבע רוויים, כל סיבוב מהיר מהקודם, הכיתה מצביעה ביד לכיוון
  הצבע שנקרא, התורן לוחץ על הריבוע הנכון בזמן.
- **🎉 מסיבת הקונפטי** (חובה #10) — כל יום שישי/שבת, ~20-40 שנ' קבועות, לא תחרותי, סיכום שבועי
  + קונפטי ענק, ה"קופאי/ת" השבועי/ת מוצג/ת כמנחה.

רוטציה בין 3 המשחקים הראשונים אוטומטית לפי מספר-יום-מוחלט (לא רק יום-בשבוע קבוע) כדי שאותו יום
בשבוע לא יקבל תמיד את אותו משחק שבוע אחרי שבוע (מונע את מלכודת "חזרתיות ללא שינוי", RESEARCH §מלכודות
#1).

## 7. בדיקת תקינות

```
node --check js/class-games.js
```
