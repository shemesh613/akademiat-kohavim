# אקדמיית כוכבים — הרחבה: האי הגדול + משחק כיתתי
## חוזה ממשק משותף (SPEC) — כל הסוכנים חייבים לציית לו

## 0. הקשר
האפליקציה הקיימת: `index.html` (189KB, single file) — מערכת נקודות כיתתית.
- מצב מורה: `?t=shay2026nikud` | מצב תלמיד: URL נקי
- state נשמר ב-`localStorage['cps_state_v2']` + Firebase RTDB
  `https://nikud-7eb2e-default-rtdb.europe-west1.firebasedatabase.app/cps_v1.json`
- Three.js r128 גלובלי (`THREE`), GLTFLoader טעון.
- state schema: `{ classes:[{id,name,students:[{id,name,column,points,weeklyPoints,equipped,owned,stickers,achievements,streak}]}], activeClassId, pending:[] }`

## 1. ארכיטקטורה — קבצים נפרדים (חובה!)
כל סוכן כותב **רק את הקובץ שלו**. אסור לגעת ב-`index.html` (האינטגרציה נעשית ע"י המתאם).

## 2. ה-API הגלובלי המשותף (`window.AK`)
המתאם יוסיף ל-index.html את השכבה הזו. **כל סוכן מניח שהיא קיימת:**

```js
window.AK = {
  state,                       // ה-state הגלובלי החי
  save(),                      // שומר ל-localStorage + ענן
  getActiveClass(),            // {id,name,students:[]}
  findStudent(id),             // {student, klass} | null
  isTeacher: Boolean,          // APP_MODE === 'teacher'
  playSound(type),             // 'coin'|'rankup'|'buy'|'error'|'combo'
  burstConfetti(x,y,count),
  toast(msg),                  // הודעה קופצת
  escapeHtml(s),
};
```

### state חדש — נוסף ל-class (לא לתלמיד!)
```js
klass.island = {
  coins: 0,             // "אבני בנייה" של הכיתה — נצברות מנקודות התלמידים
  spent: 0,
  regions: ['beach'],   // אזורים שנפתחו
  items: [              // מבנים שהוצבו
    { id:'palm', r:'beach', x:3, z:-2, rot:0, by:'s_xxx', t:1690000000000 }
  ],
  level: 1,             // רמת האי 1..40
  history: [],          // אירועים אחרונים (max 50)
};
klass.games = {
  lastPlayed: 0,        // timestamp
  totalScore: 0,        // נקודות שהמשחקים הכניסו לקופת האי
  plays: [],            // {gameId, studentId, score, t} — max 100
  dailyDone: false,
};
```
**כלל זהב:** לעולם לא לשנות את `student.points` הקיים ולא לשבור מיגרציות. כל שדה חדש
חייב fallback: `klass.island = klass.island || {...}`.

## 3. כלכלה — מתוכננת לשנת לימודים שלמה (~36 שבועות)
- כיתה של 24 תלמידים צוברת ~60–120 נקודות ליום → ~400–600 לשבוע.
- **המרה:** כל 10 נקודות שתלמיד מקבל = 1 "אבן בנייה" (`coins`) לקופת הכיתה.
  (המרה אוטומטית ב-hook על `addPoints`, לא גורעת מהתלמיד!)
- מטרה: ~50 אבנים/שבוע → ~1,800 לשנה. תמחור המבנים והאזורים חייב להתאים לזה.
- **פתיחת אזורים** (`level` / `coins` מצטבר `coins+spent`):
  | # | אזור | שם | נפתח ב | תקופה משוערת |
  |---|---|---|---|---|
  | 1 | `beach` | חוף הכוכבים | 0 | ספטמבר |
  | 2 | `forest` | יער הלחישות | 120 | אוקטובר |
  | 3 | `farm` | חוות האלופים | 300 | נובמבר |
  | 4 | `village` | כפר הידע | 520 | דצמבר-ינואר |
  | 5 | `mountain` | הר הקרח | 780 | פברואר |
  | 6 | `desert` | מדבר הזהב | 1050 | מרץ |
  | 7 | `volcano` | הר האש | 1350 | אפריל-מאי |
  | 8 | `sky` | איי השמיים | 1700 | יוני |

## 4. חוקי איכות ויזואלית (חובה על כולם)
- סגנון: **voxel/low-poly צבעוני** — כמו merotz-haalufim אבל **הרבה יותר מפורט**.
- כל מבנה: 8–25 meshes, לפחות 3 צבעים, פרט אחד מונפש (מסתובב/מרחף/מהבהב).
- אנטי-אליאסינג, צללים (PCFSoft), fog, שמיים עם gradient.
- ביצועים: יעד 60fps על טאבלט. תקציב: <15,000 משולשים לאזור פעיל.
  להשתמש ב-`InstancedMesh` לאלמנטים חוזרים (עשב, סלעים).
- עברית RTL, פונט קיים באפליקציה. אמוג'ים כן.
- כל טקסט בעברית.

## 5. שני אילוצי-על — גוברים על כל שיקול אחר

### 5.1 אפס עומס למורה
המורה לא מנהל שום דבר. הכל אוטומטי:
- המרת נקודות ל-`island.coins` — אוטומטית ב-hook, בלי קליק.
- פתיחת אזור חדש — אוטומטית כשעוברים סף, עם אנימציית חשיפה.
- בחירת שחקן היום למשחק — האפליקציה בוחרת ומכריזה.
- שמירה, ניקוד, הכרזות — אוטומטי.
- **תקציב מותר למורה: קליק אחד להפעלה. לא יותר.**
- אפשר להעביר תפקידים לילדים (תורן משחק / ועדת אי) — זה גם מנוע מוטיבציה.

### 5.2 הכל רץ על המקרן הכיתתי
מסך אחד גדול, צפייה ממרחק 6–8 מטר, קלט יחיד (עכבר/מקלדת אחד).
- **טקסט ענק**: כותרות ≥48px, טקסט ראשי ≥28px, שום דבר מתחת ל-22px.
- ניגודיות גבוהה מאוד — מקרנים דהויים "אוכלים" צבעים חלשים ואפורים דקים.
- לא יותר מ-7 אלמנטים על המסך בו-זמנית. מעט מילים, הרבה ויזואל.
- אנימציות גדולות וברורות שנראות מהשורה האחורית.
- מצב `?projector=1` — פריסת ענק, בלי כפתורי ניהול.
- האי צריך לעבוד גם כ**מסך רקע (ambient)** שרץ בשקט במהלך היום.

## 5.3 מנגנון הליבה: **זמן משחק נקנה בנקודות**
הנקודות של הכיתה הן לא רק כסף לבנייה — הן **שניות משחק**.
- הכיתה צוברת נקודות במהלך היום → הנקודות מיתרגמות ל**שניות** בקופת הזמן.
- בהפסקה: כמה זמן המשחק יימשך? בדיוק כמה שהכיתה הרוויחה היום.
  יום חלש = 25 שניות. יום מצוין = 90 שניות. **המסך מראה את זה לפני שמתחילים.**
- זה הופך את הנקודות למוחשיות בזמן אמת: כל תשובה טובה בשיעור = עוד שנייה של כיף בהפסקה,
  והכיתה כולה מרוויחה מזה — לחץ חברתי חיובי, לא תחרות בין ילדים.
- טיימר ענק על המקרן שמתמלא/מתרוקן — הדבר הכי דרמטי במסך.
- ניתן להרוויח שניות בונוס תוך כדי המשחק (הצופים משיגים אותן) → הארכת הזמן בשידור חי.
- **חובה תקרה ורצפה:** מינימום שמובטח תמיד (שלא ייווצר יום בלי משחק בכלל → עונש קולקטיבי
  שיפגע במוטיבציה) ומקסימום (שלא יבלע את ההפסקה). המחקר יקבע מספרים.
- **⚠️ לא להגזים עם הזמן!** זו הפסקת אוכל, לא שיעור. תקרה קשיחה:
  **המשחק עצמו ≤60 שניות, וכל האירוע מקצה לקצה ≤2 דקות** (הכרזה + משחק + חגיגה).
  עדיף "רוצים עוד!" מאשר ילדים משתעממים. הזמן שנצבר מעבר לתקרה **לא הולך לאיבוד** —
  הוא מתגלגל לקופת האי או לסבב נוסף ביום אחר.
